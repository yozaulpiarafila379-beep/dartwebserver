Object Oriented Programming (OOP) - Tema Siswa SMA

Object Oriented Programming (OOP) adalah paradigma pemrograman yang berfokus pada objek sebagai unit utama. Setiap objek memiliki state (data) dan behavior (perilaku). Pada tugas ini, konsep OOP diterapkan pada kasus data siswa SMA untuk membuat kode yang modular, reusable, dan mudah dipelihara.

1. Class & Object (Jurusan)

Contoh pendefinisian class Jurusan dengan dua atribut (namaJurusan dan jumlahSiswa) serta dua method (tampilkanInfo dan tambahSiswa).

Contoh Kode 1: Class Jurusan dengan Metode Dasar

class Jurusan {
  String? namaJurusan;
  int? jumlahSiswa;

  void tampilkanInfo() {
    print('Informasi Jurusan!');
  }

  void tambahSiswa() {
    print('Siswa baru ditambahkan ke jurusan!');
  }
}

void main() {
  Jurusan jurusanKu = Jurusan();
  jurusanKu.namaJurusan = "MIPA";
  jurusanKu.jumlahSiswa = 120;

  jurusanKu.tampilkanInfo();
  jurusanKu.tambahSiswa();
}

Contoh Kode 2: Class Jurusan dengan Menampilkan Properti

class Jurusan {
  String? namaJurusan;
  int? jumlahSiswa;
}

void main() {
  Jurusan jurusanKu = Jurusan();
  jurusanKu.namaJurusan = "MIPA";
  jurusanKu.jumlahSiswa = 120;

  print(jurusanKu.namaJurusan);
  print(jurusanKu.jumlahSiswa);
}

Contoh Kode 3: Class Jurusan dengan Konstruktor Posisional

class Jurusan {
  String namaJurusan;
  int jumlahSiswa;

  Jurusan(this.namaJurusan, this.jumlahSiswa);

  void tampilkanInfo() {
    print('Jurusan Aktif: $namaJurusan');
  }
}

void main() {
  Jurusan jurusanKu = Jurusan("MIPA", 120);

  jurusanKu.tampilkanInfo();
  print(jurusanKu.jumlahSiswa);
}

2. Contoh Kelas Lain dalam OOP (Tema Siswa)

Contoh Kode 4: Class Siswa

class Siswa {
  String nama;
  String nis;
  double nilaiRataRata;

  Siswa(this.nama, this.nis, this.nilaiRataRata);

  void tampilkanData() {
    print('Nama : $nama');
    print('NIS : $nis');
    print('Nilai Rata-Rata : $nilaiRataRata');
  }
}

void main() {
  Siswa siswa1 = Siswa("Yoza Ulpia Rafila", "10251410", 89.25);
  siswa1.tampilkanData();
}

Contoh Kode 5: Class IndividuSiswa dengan Properti Tersembunyi

class IndividuSiswa {
  String _namaLengkap;
  int usia;

  IndividuSiswa(this._namaLengkap, this.usia);

  void perbaruiNama(String namaBaru) => _namaLengkap = namaBaru;

  String ambilNama() => _namaLengkap;
}

void main() {
  IndividuSiswa siswa = IndividuSiswa("Yoza Ulpia Rafila", 17);

  print(siswa.usia);
  siswa.perbaruiNama("Yoza U. Rafila");
  print(siswa.ambilNama());
}

Contoh Kode 6: Class AlatTulis dengan Konstruktor Sederhana

class AlatTulis {
  String namaBarang;

  AlatTulis(this.namaBarang);
}

void main() {
  AlatTulis barangSiswa = AlatTulis("Pulpen Mekanik Yoza");
  print(barangSiswa.namaBarang);
}

Contoh Kode 7: Class BukuPelajaran dengan Konstruktor Bernama

class BukuPelajaran {
  final String kodeBuku, namaBuku;
  final double harga;
  final int jumlahHalaman;
  final String mataPelajaran;

  BukuPelajaran({
    required this.kodeBuku,
    required this.namaBuku,
    required this.harga,
    this.jumlahHalaman = 100,
    this.mataPelajaran = "Umum",
  });
}

void main() {
  BukuPelajaran matematika = BukuPelajaran(
    kodeBuku: "B001",
    namaBuku: "Matematika SMA Kelas XI",
    harga: 75000,
    jumlahHalaman: 150,
    mataPelajaran: "Matematika",
  );

  BukuPelajaran fisika = BukuPelajaran(
    kodeBuku: "B002",
    namaBuku: "Fisika SMA Kelas XI",
    harga: 80000,
    jumlahHalaman: 160,
  );

  BukuPelajaran bahasaInggris = BukuPelajaran(
    kodeBuku: "B003",
    namaBuku: "Bahasa Inggris SMA",
    harga: 65000,
  );

  print(matematika.namaBuku);
  print(fisika.namaBuku);
  print(bahasaInggris.namaBuku);
}

Kesimpulan

Dari berbagai contoh kode dengan tema siswa SMA yang telah ditampilkan, dapat disimpulkan bahwa Object Oriented Programming (OOP) memudahkan pengelolaan kode dengan mengelompokkan data dan perilaku ke dalam kelas. Setiap kelas dapat memiliki atribut untuk menyimpan informasi siswa, jurusan, atau barang terkait sekolah, serta method untuk menjalankan aksi yang relevan.

Konsep seperti konstruktor (posisional dan bernama) membantu dalam membuat objek dengan cara yang lebih fleksibel, sedangkan properti tersembunyi membantu dalam melindungi data agar tidak diakses secara sembarangan.
